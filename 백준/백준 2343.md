# 기타 레슨

## 문제 분석

- N (1 ≤ N ≤ 100,000), M (1 ≤ M ≤ N)이기 때문에 각 강의 그룹의 값을 구하면 시간복잡도가 많이 든다.
- 블루레이의 크기는 모두 똑같기 때문에 블루레이 크기를 이진탐색으로 구해서, 각 강의를 모두 담을 수 있을지 확인한다.

## 문제풀이

- M=N 인 경우, 강의 중 최대 크기가 블루레이 크기가 된다.
- 따라서 블루레이의 크기 중 가장 작은 값: 가장 큰 강의시간 값
- M=1 인 경우, 강의시간을 모두 합한 값이 블루레이 크기가 된다.
- 따라서 블루레이 크기 중 가장 큰 값: 강의시간을 모두 합한 값

### 이진탐색

- cnt: 위에서 정한 범위 안에서 블루레이 크기 중간값을 기준으로 나오는 블루레이 개수
- cnt > M : 블루레이 크기가 더 커져야 하기 때문에 더 큰 범위 체크

``` java

        while (start <= end) {
            int sum = 0;
            int cnt = 0;
            int mid = (start + end) / 2;

            for (int i = 0; i < arr.length; i++) {
                if (sum + arr[i] > mid) {
                    sum = 0;
                    cnt++;
                }
                sum += arr[i];
            }

            if (sum != 0) {
                cnt++;
            }

            if (cnt > M) {
                start = mid + 1;
            } else {
                end = mid - 1;
            }
        }

```
